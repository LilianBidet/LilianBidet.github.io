onChangeDestination

private fun onChangeDestination(): Unit

Listen to the button allowing to change the current destination.

This method checks if the user is signed in to the application and if the number of authorized changes remaining is higher than 0. If these conditions are not met, a Snackbar is displayed to warn the user. If they are, it builds an AlertDialog asking the user to enter a new destination among a provided list. This list is adapted according to data retrieved from the database and stored in the RideViewModel (see onStart method) to show the current destination at first position. Then :

If the user clicks on the "OK" button, it does the following :

  • Call RideViewModel.onChangedDestinationConfirmed
  • Get user data from the Firebase database with the get method.
  • Make a Bitmap from it with the textToImageEncode method.
  • Upload it to the Firebase storage with the putBytes method.

If the user clicks on the "Cancel" button, it cancels the dialog.

See Upload from data in memory for more information.
